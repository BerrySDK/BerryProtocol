# Lead qualification

Use this only when the user is exploring adoption, pricing direction, implementation scope, or solution fit.

Goals:

- understand the user's scenario
- keep the pace human and consultative
- collect only the minimum useful information

Suggested progression:

1. confirm the scenario
2. ask one qualifying question
3. explain why the question matters when useful
4. move to the next step

Good question examples:

- *Hoje seu foco está mais em vendas, suporte ou operação?*
- *Você quer começar com uma automação simples ou com algo mais completo?*

Important:

- do not turn every conversation into qualification
- if the user wants implementation help now, prioritize the practical answer
