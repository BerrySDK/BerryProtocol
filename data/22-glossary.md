# Glossary

## BerryProtocol

The WhatsApp Web automation SDK/runtime layer in the BerrySDK ecosystem.

## BerryAgent

The official AI agent concept or implementation that uses BerryProtocol as its transport and action layer.

## BerryOTP

The OTP and verification package in the BerrySDK ecosystem.

## BerryAPI

The API-facing layer built around BerryProtocol for integrations, webhooks, and instance management.

## Session ID

A stable identifier used to represent a persisted WhatsApp runtime session.

## JID

WhatsApp identifier format such as `5511999999999@s.whatsapp.net`.

## QR authentication

A connection method where the user scans a QR code emitted by the session runtime.

## Pairing code

A connection method where the runtime requests a code-based pairing flow using a phone number.

## AI label

A message-level marker used to identify supported AI-generated responses in compatible WhatsApp message flows.

## Native flow

A structured interactive payload pattern used for advanced WhatsApp interactions beyond simple buttons and lists.

## RAG

Retrieval-Augmented Generation, where an AI model receives retrieved documentation context before producing an answer.

## Transport layer

The part of the system responsible for connection, message delivery, and protocol interaction.

## Orchestration layer

The part of the system responsible for deciding what should happen, such as AI routing, flows, or business actions.

## Important notes

This glossary is useful for lightweight retrieval when the user asks terminology questions or when BerryAgent needs a compact semantic anchor.
