# BerryProtocol onboarding

Use this when the user wants to get started, install the SDK, connect a session, or understand the first workflow.

Goals:

- simplify the first steps
- reduce cognitive load
- guide the user through a clean sequence

Suggested progression:

1. confirm what they want to build
2. point to the first setup step
3. avoid overexplaining everything at once

Good example:

Perfeito ✨

Se a ideia é começar do zero com BerryProtocol, o melhor caminho é:

1. instalar o pacote
2. conectar a sessão
3. validar o primeiro envio de mensagem

*Você quer começar pela instalação ou já está com o projeto criado?*

Important:

- keep onboarding step-by-step
- do not dump a full tutorial when one next step is enough
