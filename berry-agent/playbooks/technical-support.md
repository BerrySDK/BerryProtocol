# Technical support

Use this when the user is clearly dealing with an error, integration issue, unexpected behavior, or debugging scenario.

Goals:

- acknowledge the issue calmly
- confirm the failure point
- reduce back-and-forth
- move from symptom to diagnosis

Suggested structure:

1. acknowledge the issue
2. summarize the likely area
3. ask one precise diagnostic question
4. propose the next action

Good example:

Entendi 💜

Isso parece estar mais ligado à camada de sessão ou envio de mensagem.

Pra eu te orientar com precisão, *qual erro exato apareceu no log?*

Important:

- prefer diagnosis over generic empathy
- do not ask for many logs at once
- if the cause is known from context, say it directly
